My colleague Eli and I recently attended [PyConZA](<https://za.pycon.org/#>) to share how we use [Python](<https://www.python.org/>) in our work. It was a hybrid event with an in-person audience of more than 50 people at the prestigious Premier Splendid Inn Umhlanga, which offered nice aerial views from the 10th floor.

![](/img/blog/erpnext/dpQcpMx.jpg)

The main concepts covered by the talks can be broadly categorised as:

  1. Use of Python in managing databases (PostgreSQL)
  2. Processing data pipelines
  3. Do's and Don'ts for building applications using Python.

I also had the privilege of presenting a talk, whose title was [Leveraging Python in GeoSpatial Applications](<https://za.pycon.org/talks/26-leveraging-python-in-geospatial-applications/>). I went over how we at Kartoza use Python in the geospatial industry and was based on some the work we have done on previous projects such as the [SAGTA MapDownloader](<https://sagtamapdownloader.kartoza.com/>) and SDIs (Spatial Data Infrastructure) like one we did recently for managing electrical infrastructure in ECOWAS.

The key things I covered were:

  1. How to automate loading data from various sources into a single normalised database using Python scripts in QGIS.
  2. How to create custom Python functions to aid in creating form constraints in QGIS.
  3. How to use Python to create actions to automate processes in QGIS and expose these in the attribute table in QGIS.
  4. A practical demonstration of how to set up QGIS forms and and of the workflow for capturing data from a mobile device using MerginMaps and Mergin DB Sync, and how this synchronises with the GeoNode-based SDI

Python as a programming language is very powerful and can be used in many ways as noted in some of the lightning talks where, for example, a user is developing an application to monitor his garage doors and other household components.
